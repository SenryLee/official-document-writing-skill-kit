# 意图判断与交互引导

用于判断用户到底要生成、修改还是审查公文，并控制提问成本。

## 目录

- 任务类型判断
- 三轮以内交互协议
- 信息明确时直接处理
- 只给主题时的处理方式
- 文种判断表
- 可默认假设

## 1. 任务类型判断

| 类型 | 用户常见表达 | 后续读取 |
|---|---|---|
| 生成 | 写一份、起草、生成、拟一篇、给我做个初稿、根据材料写 | `generate-workflow.md` |
| 修改 | 修改、润色、改正式、提高度、压缩、扩写、改标题、重写 | `revise-workflow.md` |
| 审查 | 审查、把关、看看问题、能不能用、是否合格、按标准验收 | `review-checklist.md` |

用户同时要求多件事时，顺序为：先审查现状，再修改或生成，最后自检。

## 2. 三轮以内交互协议

目标是低成本确认意图，不做复杂问卷。

### 第 0 步：判断是否可以直接做

出现以下情况，直接处理，不追问：

- 用户给了参考资料，并说“按照参考资料写”。
- 用户给了原稿，并说“帮我修改/润色/审查”。
- 用户明确了文种、主题、用途。
- 缺少的信息可以用 `[待补充]` 标记，不影响初稿方向。

### 第 1 轮：只确认方向

只有文种或用途不清时才问。最多问 3 个：

- 这份材料主要给谁看：上级、下级、领导、内部、媒体还是群众？
- 你想要的文种是工作简报、总结、通知、方案、讲话，还是我来判断？
- 用途是上报成绩、部署工作、会议发言、宣传发布，还是存档备查？

### 第 2 轮：只补核心事实

只有材料太少、无法写出实质内容时才问。最多问 3 个：

- 已经做了哪些关键工作？
- 有没有必须出现的上级文件、会议精神或固定提法？
- 有没有数字、成果、问题或下一步安排？

### 第 3 轮：只确认输出要求

只有篇幅或语气会明显影响产出时才问。最多问 2 个：

- 需要多少字左右？
- 语气偏正式书面、领导讲话、宣传报道，还是简洁实用？

第 3 轮后必须开始产出。仍缺失的信息用 `[待补充]` 标记。

## 3. 只给主题时的处理方式

用户只给主题时，先推断最可能的文种：

- “某项工作开展情况”“阶段成效” -> 工作简报或工作总结。
- “下一步怎么干”“明年工作” -> 工作计划或下一步打算。
- “要求各单位落实” -> 工作通知或工作方案。
- “领导在会上讲” -> 领导讲话。
- “会后形成材料” -> 会议纪要。
- “宣传某项工作” -> 新闻稿、通讯稿或工作简报。
- “推荐某个人” -> 先进事迹。
- “查摆问题整改” -> 民主生活会对照检查材料。

可直接这样开头：

`我先按[文种]处理，假设用途为[用途]、对象为[受众]。缺少事实处用[待补充]标记。`

## 4. 文种判断表

| 场景 | 优先文种 | 判断要点 |
|---|---|---|
| 阶段性工作成绩、内部宣传 | 工作简报 | 短、集中、突出措施和成绩 |
| 未来工作安排 | 工作计划 | 只写未来，少写精准数据 |
| 阶段性回顾 | 工作总结 | 完成情况必备，可含问题和下一步 |
| 全年正式回顾、重大会议 | 工作报告 | 更正式、更系统 |
| 学习培训后谈认识 | 心得体会 | 从认识落到行动 |
| 领导会议发言 | 领导讲话 | 先判断事前、事中、事后或特殊 |
| 上级对下级布置事项 | 工作通知 | 清单式、要求明确 |
| 专项工作如何实施 | 工作方案 | 任务、责任、步骤、保障 |
| 个人或领导履职汇报 | 述职报告 | 以上级通知要求为框架 |
| 研究经验或问题 | 调研报告 | 选题要窄，数据要足 |
| 理论阐释、署名文章 | 理论文章 | 书面化、权威化 |
| 近期会议活动调研 | 新闻稿 | 时间、地点、人物、事件 |
| 面向公众宣传成绩或人物 | 通讯稿 | 小故事、冲突、措施、成绩 |
| 表彰推荐个人 | 先进事迹 | 围绕本职成绩和数字 |
| 对照上级要求查摆问题 | 对照检查材料 | 问题、原因、整改一一对应 |
| 党性教育 | 党课 | 理论、工作、要求结合 |
| 会议后正式摘要 | 会议纪要 | 记录决定，不记流水账 |

## 5. 可默认假设

可以默认：

- 没有特别说明时，用正式书面语。
- 没有字数要求时，先写中等篇幅。
- 没有单位历史格式时，用通用框架。
- 没有数据时，用 `[待补充数据]`。
- 没有上级文件名称时，用 `[上级文件/会议精神]`，不虚构名称。
